# Blendersynth

一个用于通过程序随机化和批量渲染生成合成数据集的Blender扩展。该插件使用户能够创建具有受控变化的多样化图像数据集，包括对象放置、材料属性、照明条件和场景参数的变化。

此外，该扩展还支持预览功能，允许用户直接在Blender中检查管道输出和计算的标签。

该扩展适用于中小型计算机视觉项目，这些项目更看重生成的易用性，而不是像Blenderproc这样的工具的效率和通用性。

## 概述
  ![Live Preview](./resources/live_preview.png)

生成器提供了一个基础单元系统，用于在Blender中定义随机化管道。用户可以指定对象、材料和照明条件如何在多个渲染中变化，然后自动输出图像及对应的元数据，适合计算机视觉训练数据集使用。用户还可以指定命名类别并将类别分配给Blender对象。多个对象可以组合在一起形成多对象实体，扩展将其分类为单个对象。

## 功能特性

- **基于分布的随机化**：使用预设分布和（正在开发）可视化节点编辑器为场景参数定义概率分布
- **管道配置**：通过连接对象变换、材料属性和照明的操作来构建随机化管道
- **批量渲染**：使用初始种子自动渲染多个具有不同场景参数的图像，以实现可重复性
- **JSON配置**：将管道配置保存和加载为JSON格式，便于版本控制和可重复性
- **多参数支持**：
  - 对象位置、旋转和缩放
  - 材料纹理和属性（粗糙度、金属度等）
  - 可见性和遮挡约束
  - 照明强度、色温和颜色
  - 相机参数和位置，相机绕中心点锁定0
  - 地平面和边界约束（正在开发）
- **导出集成**：以与计算机视觉框架兼容的格式导出渲染数据集（YOLO、COCO等）

- **实时预览**：允许用户直接在Blender中生成Blender输出的预览，包括边界框、标签和可见性统计

- **日志系统**（正在开发）：允许分析单个随机过程的结果，以检查生成和约束求解器的输出（正在开发）

## 系统要求

- **Blender**：4.5.0或更高版本
- **操作系统**：Windows、macOS或Linux

## 安装

1. 下载仓库，选择/ext/扩展存档并将其压缩为`.zip`存档。
2. 在Blender中，导航至**编辑 → 偏好设置 → 扩展程序**
3. 点击**安装...**并选择下载的`.zip`文件
4. 在"Random Dataset Generator"旁的复选框中启用该插件

该插件将在3D视口侧栏的**Synthetic**面板中出现。

## 快速入门

### 1. 创建类别和实体

分布定义参数在渲染过程中如何变化。通过随机化器面板访问分布编辑器来为参数创建分布。
```
分布编辑器 → 添加节点 → 配置参数
```

### 2. 构建管道

在管道编辑器中连接操作，以指定应随机化哪些对象和属性。
```
管道标签 → 添加操作 → 配置操作 → 连接到分布
```

### 3. 配置参数

指定受每个操作影响的对象、材料和属性。每个操作可以针对：
- 特定对象或材料槽
- 定义的照明装置
- 相机位置
- 约束参数

### 4. 生成数据集

管道配置完成后，指定从分布中提取的样本数量和标签格式来渲染数据集：
```
生成标签 → 设置输出目录 → 设置帧数 → 渲染
```

输出保存为：
- 渲染图像（PNG/EXR格式）
- 元数据txt/Json文件（每帧一个），具体格式取决于标签格式
- 配置快照和日志（用于可重复性）

渲染设置在生成时从当前Blender设置中提取。

## 架构

该扩展被组织成逻辑模块：
```
ext/
├── distribution/      # 分布节点系统和评估
├── pipeline/          # 管道数据结构和操作
├── operators/         # Blender操作符（UI交互）
├── ui/                # 用户界面面板和查看器
├── core/              # 核心渲染和生成逻辑
├── utils/             # 日志和实用函数
└── constants.py       # 配置常数
```
更详细的描述见/docs/Architecture Overview.md文件。

## 关键概念

### 分布
分布为随机化定义概率函数。提供两种模式来随机化大多数操作：使用预设分布（正态、指数等）或通过基于节点的编辑器生成自定义分布（仍在完善中）。编辑器允许节点的互联：
- **常数**：固定值
- **连续分布**（正态、均匀等）：采样概率分布
- **离散分布**：从预定义值中选择
- **选择器**：在输入中随机选择，允许复杂的多峰分布定义

### 管道操作
操作对场景元素应用随机化：
- **变换**：位置、旋转、缩放
- **材料**：纹理和属性随机化
- **照明**：光照强度、颜色、色温
- **相机**：位置和方向
- **约束**：重叠、遮挡、距离规则（正在开发）

### 配置
管道可以保存和加载为JSON格式，使以下操作成为可能：
- 实验配置的版本控制
- 可重复的数据集生成
- 轻松进行参数调整和迭代

## 使用模式

### 基本工作流程
1. 使用对象、材料和照明设置Blender场景
2. 创建一组类别和实体
3. 构建一个管道，串联场景的随机变化
4. 在Blender中配置约束和渲染设置，在扩展中配置标签设置
5. 渲染数据集并获得文本标签

### 高级工作流程
- **多阶段随机化**：链接多个操作以创建复杂的变化
- **受约束的随机化**：使用重叠、遮挡和距离约束来确保有效的配置（正在开发）
- **条件变化**：使用选择器操作和文件夹在不同的随机化策略之间选择
- **参数扫描**：创建多个管道变体以探索设计空间

## 限制

- 复杂的着色器网络可能无法通过UI完全随机化；高级情况下可能需要直接编辑材料节点
  （其思想是随机化着色器中影响完整Cycles材料外观的某些节点）
- 包含许多操作的非常大型管道可能会影响交互性能，建议使用EEVEE进行预览。
